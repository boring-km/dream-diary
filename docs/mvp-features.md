> 출처: 기획문서(`web-test-pages/dream-archive-plan.html`) + 기술설계(`docs/technical-spec.md`).
> 작성일: 2026-06-06

## 0. MVP 한 줄

> 어두운 새벽에도 부담 없이 꿈을 **빠르게 붙잡아 보관**하고, 쌓인 꿈을 **다시 들여다보는** 한국어 꿈 아카이브.

기록·검색·보관은 전부 무료. AI 해석/결제는 다음 단계. MVP는 **순수 기록 제품**.

## 1. 기능 목록 (MVP 범위)

| # | 기능 | 한 줄 | 우선순위 |
|---|---|---|---|
| F1 | 인증 (소셜 로그인) | Google · Apple 로그인 | P0 필수 |
| F2 | 꿈 기록 (CRUD) | 본문만 필수. 생성·수정·삭제·상세 한 덩어리 | P0 필수 |

## 2. 기능 상세

### F1. 인증 — 소셜 로그인  *(P0)*

- **Google + Apple만.** 둘 다 Supabase 네이티브 OAuth.
- 로그인 → `auth.users` 생성 → 트리거로 `public.users` 프로필 동기 생성.
- 비로그인 시 기록/목록 라우트 → 로그인 페이지 리다이렉트 (미들웨어).
- RLS `user_id = auth.uid()` → 본인 데이터만 접근.
- Apple 이메일 비공개(릴레이) 대비 `email` nullable.

### F2. 꿈 기록 (CRUD)  *(P0, 제품 핵심)*

꿈 하나 = `dreams` 행 하나. 생성·조회(상세)·수정·삭제가 한 덩어리. 깬 직후 몇 분이 골든타임 → **기록 마찰 최소화**가 제품 존재 이유.

**최대한 단순한 기록**
- **본문(`content`)만 필수.** 나머지 전부 선택, 나중에 추가 가능.
- 화면 열면 바로 입력 칸 → 쓰고 → 저장. 단계·모달·필수항목 강요 없음.
- **다크모드 에디터** — 어둡고 반쯤 잠든 상태 전제.
- 저장 후 언제든 재편집 = "붙잡기 → 나중에 다듬기" 2단계를 별도 모드 없이 충족.

**필드 (전부 선택)**
- **제목(`title`)**: 비면 날짜로 자동.
- **꿈 꾼 날(`dreamed_at`)**: `created_at`(기록한 날)과 분리. 새벽 기록을 어제 꿈으로.
- **감정(`emotion`)**: 단일 선택. enum `fear`(무서움) `funny`(웃김) `sad`(슬픔) `strange`(이상함) `peaceful`(평온) `anxious`(불안) `happy`(기쁨).

**저장/접근**
- 브라우저 → Supabase 직접 (RLS 본인 데이터만).
- `status` P1 기본 `private` (공개/전시관은 P3).

### F3. 목록 · 타임라인  *(P0)*

- **시간순 목록이 주 뷰** — 쌓인 꿈을 되짚음.
- **달력 보기는 보조** — 곁들이는 뷰, 날짜별 점/표시.
- 무한 스크롤 또는 페이지네이션 (구현 시 확정).

### F4. 내보내기  *(P1)*

- 본인 꿈 데이터 export — **무료**.
- 형식: JSON 또는 Markdown (구현 시 확정).

## 3. MVP 명시적 제외 (Out of Scope)

| 제외 | 이동 단계 | 비고 |
|---|---|---|
| AI 해석 (해몽/심리/감정 렌즈) | P2+ | 서버 API Route + Claude. 스키마(`dream_analyses`)만 P1 확보 |
| 결제 (Toss 구독·크레딧) | P2+ | 스키마(`tier`/`credits`)만 P1 확보 |
| 음성 기록 · 자동 전사 | P2 | 모바일 앱 (Flutter) |
| 위젯 1탭 녹음 · 리마인더 알림 | P2 | 모바일 |
| 공개 전시관 · 반응 · 신고 | P3 | 소셜 |
| AI 이미지 생성 · 꿈 갤러리 | P3 | |
| 카카오/네이버 로그인 | P2+ | MVP는 Google/Apple만 |
| 검색 · 태그 (분류·필터) | P2+ | 스키마(`tags`/`dream_tags`)만 P1 확보 |

## 4. MVP 데이터 의존 (참고)

핵심 테이블: `users`, `dreams`.
미사용(스키마만 확보): `tags`/`dream_tags`(검색·태그), `dream_analyses`(AI), `reactions`/`reports`(전시관).
상세: `docs/technical-spec.md` §3.

---

## 5. MVP 사용자 흐름 (AI 제외판)

1. **소셜 로그인** (Google/Apple)
2. **새벽 — 붙잡기**: 다크 에디터에 떠오르는 대로 기록 → 저장
3. **아침 — 다듬기**: 제목·감정 추가
4. **한 달 뒤 — 되짚기**: 목록/타임라인으로 반복 상징·감정의 결 탐색
5. (선택) **내보내기**로 본인 아카이브 백업
