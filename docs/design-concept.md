## 0. 한 줄

> **꿈을 적으면 손그림 종이 카드 한 장을 받고, 그게 서랍에 쌓여 — 나중에 흩어진 더미에서 잊은 꿈을 "발굴"하는 담백한 아카이브.**

생산성 도구 아님. **추억 상자.** 검색 효율이 아니라 발굴의 정서가 핵심.

## 1. 제품 DNA

| 축 | 결정 |
|---|---|
| 메타포 | **서랍 단독** (4개 감정 서랍, 위치=깊이=비밀도) |
| 철학 | **발굴형 / 안티-검색** (의도된 비효율 = 정체성) |
| 내용물 | 서랍 속 흩어진 손그림 종이 카드 더미 (마구잡이, id 기반 고정 배치) |
| 카드 | 손그림 + **고유번호** + 그날의 손그림 도장 아이콘 |
| 자기보상 | "오늘의 발굴"(랜덤 1장), 수집 카운터·뱃지 |
| 검색 | 구석 작은 손그림 돋보기, 본문 매칭, 평소 안 드러남, 닫으면 흔적 0 |
| 비주얼 | 페이퍼 라이트, 파란 잉크 단색 절제, 깔끔 인쇄체 + 손그림, 미니멀 플랫 |

## 2. 메타포 결정 — 왜 서랍인가

후보를 단독 메타포로 분리 평가 → **개성 우선** 재평가 → 사용자 핵심 욕구("발굴의 재미")로 확정.

| 컨셉 | 결론 |
|---|---|
| **🗄 서랍** | **채택.** 꿈=은밀·무의식과 의미 정합 1위. "위치=심리적 깊이"가 독창적 분류 메커니즘. 닫힘·뒤적임·묻힘 = 발굴과 정합 최고 |
| 🎫 우표/우표첩 | 탈락(단독). 비주얼·수집·손그림 강하나 "전시·정렬"이라 **발굴과 반대**. 단 카드 손그림 스타일은 차용 |
| 🌱 정원/숲 | 탈락. 느좋·손그림 좋지만 식물성장 = 웰니스 앱 클리셰(개성↓) + 한눈에 보임(발굴↓) |
| 🗄 서랍의 초기 약점 | 단위 모호·확장·손그림 불일치 → 사용자가 해소(서랍=함, 고정 개수, 손그림 전역 컨셉) |
| 🏠 집 / 📮 우체통 | 탈락. 집=과대·복잡. 우체통=주고받음(소셜) → P3 "받은 평가함"으로만 부활 |

### 발굴을 재미로 만드는 선 (재미 vs 불안)
- **세렌디피티(우연한 재회)는 살리고, 상실(영영 못 찾음)은 0.**
- 안전장치: ① 맨 위 서랍 = 자주 꺼내볼 꿈(즐겨찾기) ② 숨은 돋보기 = 최후의 검색 ③ 서랍 3~4개로 거친 구획.
- 고유번호 = 본문 기억 안 나도 호명 가능한 좌표("내 37번 꿈").

## 3. 디자인 방향 (요약 — 상세는 `.impeccable.md`)

- **테마**: 페이퍼 라이트 통일 (따뜻한 오프화이트). 다크 없음.
- **색**: 파란 잉크 단색 절제 — 거의 모노. 60-30-10, 파랑 희소하게.
- **글자**: 깔끔한 인쇄체만 (손글씨체 금지). 개성은 손그림 일러스트/아이콘.
- **질감**: 미니멀 플랫. 물성은 텍스처가 아니라 **인터랙션**(서랍 열림, 카드 뒤적임)으로.
- **5원칙**: ① 발굴>검색 ② 손그림이 유일한 장식 ③ 파랑 1색+종이 ④ 담백·여백 ⑤ 물성은 동작으로.
- **안티**: 식물성장 웰니스 클리셰, AI 슬롭(그라데이션·네온·글래스), 정렬된 대시보드 느낌.

## 4. 핵심 메커니즘 — 3화면 구조

**화면 A 서랍장 → (서랍 선택) → 화면 B 열린 서랍 → (카드 클릭) → 화면 C 꿈 상세**

1. **기록(붙잡기)** — 메타포 0. 다크/담백 에디터에 바로 쓰기. 골든타임 마찰 최소. 저장 시 카드 발행(고유번호 +1, 그날 아이콘 자동 부여, 감정→서랍 자동 배정).
2. **화면 A — 서랍장** — 4개 감정 서랍 UI(닫힘). 위치=깊이=비밀도. 서랍 선택 → 화면 B로 전환(서랍 열림).
3. **화면 B — 열린 서랍(별도 화면)** — 진입 시 **손그림 카드가 한 번 뒤섞이는 애니메이션**. 카드는 불투명, 겹쳐 흩어짐. **카드 = 손그림 + 고유번호만**(본문 없음). **드래그로 꺼내기.** "오늘의 발굴" 랜덤 1장.
4. **화면 C — 꿈 상세** — 카드 클릭 시에만 본문 표시(별도 화면). 본문·감정·날짜·고유번호·아이콘.
5. **검색(비상구)** — 구석 손그림 돋보기. 본문 매칭. 평소 안 보임, 결과도 발굴 톤, 닫으면 흔적 0.
6. **자기보상** — 수집 카운터(고유번호=몇 개 모았나), 뱃지(연속·횟수), 담백한 한 문장 피드백.

## 5. 기능 로드맵 (발굴형으로 갱신)

기존 P1(빌드 완료: 인증·CRUD·목록·export) 위에 컨셉을 얹고, P2/P3는 기존 기술 spec과 정합.

| 단계 | 추가 | 비고 |
|---|---|---|
| **MVP (현재 + α)** | ✓있음: 인증·CRUD·목록·export. **+ 손그림 카드(고유번호+그날 아이콘 자동, AI 0)** · **서랍 3~4개 + 발굴 더미 뷰** · **숨은 돋보기 검색** | 카드·서랍·발굴이 제품 정체성. 비용 0으로 "그림드림"다움 확보. 아이콘은 프리셋 풀에서 감정/키워드 결정 |
| **P1.5 (백엔드無)** | "오늘의 발굴" 랜덤 · 뱃지(연속·횟수) · 감정별 카드 구분(테두리/형태) · 담백한 한 문장 피드백(템플릿·비-AI) | 자기보상 루프 완성. 피드백은 AI 전 검증 |
| **P2 (서버+AI+모바일)** | 한 문장 피드백 → 실제 Claude(해몽/심리/감정 렌즈) · 음성기록·위젯·리마인더(Flutter) · 결제 · 카드 아이콘 AI 생성(블루 라인 유지) · 하루 노래 추천 | 기존 technical-spec 그대로. 스키마 확보됨 |
| **P3 (소셜)** | 공개/비공개 전시관 · **평가문구**(당근식 표정/문구 선택, 좋은말 풀) · **우편함 = 받은 평가함**(우체통 메타포 재활용) · "내 37번 봐줘" 카드 공유 | status 스키마 이미 있음 |

## 6. 데이터 영향 (현재 스키마 대비)

- **신규 컬럼**: `dreams.dream_no int` — 유저별 순번, 영구·불변·결번 유지(삭제해도 max+1 발행). uuid는 내부 키 유지.
- **신규 컬럼**: 카드 아이콘 식별자(예: `icon_key text`) — MVP는 프리셋 풀, P2 AI 생성으로 확장.
- **서랍**: 고정 3~4개라면 enum/소형 테이블. 카드↔서랍 = `dreams.drawer text` 단일 배정.
- 기존 `emotion`/`status`/`tags` 스키마 재활용(감정 표현·공개·P2 검색).

## 7. 서랍 구성 (확정: 4개 감정 서랍 + 깊이=비밀도)

위→아래 = 자주 꺼냄→묻어둠. emotion enum 7종을 4서랍에 매핑:

| 순서 | 서랍 | 매핑 감정 | 성격 |
|---|---|---|---|
| 1 (위) | **좋은 꿈** | happy, peaceful, funny | 자주 꺼내봄 (즐겨찾기 성격) |
| 2 | **이상한 꿈** | strange | |
| 3 | **슬픈 꿈** | sad | |
| 4 (깊은 아래) | **악몽** | fear, anxious | 묻어둠 (숨김) |

- 기록 시 감정 선택 → 해당 서랍 자동 투입. 미선택 시 기본 서랍(예: 이상한 꿈) → 나중에 옮기기.
- 사용자 직접 명명은 P2로 미룸(심플 우선).
